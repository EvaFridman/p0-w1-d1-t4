# p0-w1-d1-t4 · Ноль ошибок валидатора + превью в мессенджере

> Тип: research (часть материала на лекции не давали — ищешь сама) · Репозиторий: один отдельный

## Контекст

Ты скинула ссылку на свою страницу в рабочий чат, а превью вышло уродское: без картинки, без
заголовка, просто голый URL. Заодно выяснилось, что страница не проходит валидатор. На лекции
про это не говорили — значит, идёшь и разбираешься сама, как взрослый разработчик.

```
┌────────────┐  прогон   ┌────────────────┐
│ index.html │ ────────▶ │ W3C Validator  │
└────────────┘           └───────┬────────┘
                                 │ ошибки
                                 ▼
                    чинишь до 0  +  добавляешь OG-теги (ищешь сама)
```

## Что нужно сделать

Взять свою страницу из t2 и довести её до двух состояний: (1) ноль ошибок и предупреждений в
официальном W3C-валидаторе; (2) красивое превью при вставке ссылки в мессенджер/соцсеть.
Второе требует тегов, которых на лекции не было — нужно найти, что это и как работает.

## Релизы

### Release 1 — валидация
Прогнать страницу через W3C Markup Validator, выписать все ошибки в `NOTES.md`, починить до нуля.
Для каждой ошибки — одна строка: что было не так и как поправила.

### Release 2 — мета-теги для превью
Разобраться, что такое Open Graph (`og:title`, `og:description`, `og:image`) и базовые
`meta name="description"`. Добавить их в `head`. В `NOTES.md` — короткое объяснение своими
словами: зачем эти теги и кто их читает.

### Release 3 — проверка превью
Проверить превью через любой публичный OG-debugger (найти самой). Приложить в `NOTES.md`
скриншот результата. Если картинка не подтянулась — разобраться почему (абсолютный URL,
размеры) и починить.

## Критерии приёмки

* W3C-валидатор показывает 0 ошибок и 0 предупреждений.
* В `head` есть корректные `description` и Open Graph теги.
* В `NOTES.md` своими словами объяснено, что такое OG-теги и почему URL картинки должен быть
  абсолютным.

## Подсказки

* Ищи: «W3C Markup Validation Service», «Open Graph protocol», «OG image absolute URL».
* Готовое решение копировать не надо — разберись, что делает каждый тег, иначе на ревью не
  объяснишь.
